# Human-Todo Issue Template

```
Title: [HUMAN] <short action needed>
Labels: human-todo, <priority: p0|p1|p2>

Body:

## Why this matters
<1–2 lines linking to the ₹15k goal — e.g. "blocks 25% of expected revenue path">

## What I need from you
<time estimate + steps>

## Steps (detailed)
1. ...
2. ...

## Blocking work
- PRs I cannot open until this is resolved: <list or "none">
- Alternative I can do while waiting: <fallback>

## Reply here with
- "done" (I'll pick up next session)
- "skip" (I'll drop this permanently)
- "later" (I'll snooze 14 days)
- Or a question / counter-plan
```

## Priority guide
- **p0** — blocks the goal (e.g. Razorpay not wired, Supabase credentials missing).
- **p1** — high-leverage but not blocking (e.g. GBP setup, Merchant Center, real product photography).
- **p2** — nice-to-have (e.g. new social handle, brand partnerships).

## Never open a human-todo for
- Anything you could PR yourself (write the PR instead).
- Trivial questions ("what color should the button be?"). Batch these into the session summary.
- Duplicates of open issues (check `list_open_issues` first).
